# Database Documentation

## Overview

The application uses **MongoDB** as the primary database with **Mongoose ODM** for schema management and validation.

**Connection**: 
- Development: MongoDB Atlas (free tier)
- Production: MongoDB Atlas (paid tier)
- Pool Size: 10 connections

## Collections & Models

### 1. User Collection

Stores user account information and subscription status.

**Model**: `lib/db/models/User.ts`

```typescript
{
  _id: ObjectId,
  email: String,                    // unique, lowercase, trimmed
  passwordHash: String,             // bcryptjs hashed
  isVerified: Boolean,              // email verification status
  createdAt: Date,                  // account creation date
  subscription: {
    plan: 'free' | 'pro',
    status: 'active' | 'inactive',
    expiresAt: Date?                // null if never expires
  },
  updatedAt: Date
}
```

**Indexes**:
- `email` (unique)
- `createdAt` (for sorting)

**Queries**:
```typescript
// Find user by email
const user = await UserModel.findOne({ email: userEmail });

// Update subscription
await UserModel.findByIdAndUpdate(userId, {
  'subscription.plan': 'pro',
  'subscription.status': 'active'
});

// Check email verification
const isVerified = user.isVerified;
```

**Validation**:
- Email: required, unique, valid email format
- PasswordHash: required, min 60 chars (bcryptjs)
- Subscription.plan: enum ['free', 'pro']
- Subscription.status: enum ['active', 'inactive']

### 2. Project Collection

Stores user-created writing projects.

**Model**: `lib/db/models/Project.ts`

```typescript
{
  _id: ObjectId,
  userId: ObjectId,                 // reference to User
  title: String,                    // 1-200 chars
  description: String?,             // optional, max 500 chars
  category: String,                 // from predefined list
  contentType: String,              // blog, article, report, guide, etc
  status: 'draft' | 'in_progress' | 'completed',
  createdAt: Date,
  updatedAt: Date
}
```

**Indexes**:
- `userId` + `createdAt` (for listing user's projects)
- `status` (for filtering)

**Queries**:
```typescript
// Get user's projects (with pagination)
const projects = await ProjectModel
  .find({ userId })
  .sort({ createdAt: -1 })
  .limit(20)
  .skip(offset);

// Create new project
const project = await ProjectModel.create({
  userId,
  title,
  description,
  category,
  contentType,
  status: 'draft'
});

// Update status
await ProjectModel.findByIdAndUpdate(projectId, {
  status: 'in_progress'
});
```

**Validation**:
- userId: required, must reference existing User
- title: required, 1-200 characters
- description: optional, max 500 characters
- category: required, must be in allowed categories
- contentType: required, must be in allowed types
- status: enum ['draft', 'in_progress', 'completed']

**Categories** (from `lib/config/category-config.ts`):
- Technology
- Business
- Health & Wellness
- Education
- Entertainment
- Sports
- Other

### 7. OtpCode Collection

Stores one-time passwords for email verification.

**Model**: `lib/db/models/OtpCode.ts`

```typescript
{
  _id: ObjectId,
  email: String,                    // user's email
  code: String,                     // 6-digit OTP
  expiresAt: Date,                  // 10 minutes from creation
  isUsed: Boolean,                  // verification completed?
  createdAt: Date
}
```

**Indexes**:
- `email` (for lookup)
- `expiresAt` (for cleanup)

**Queries**:
```typescript
// Generate OTP
const otp = Math.random().toString().slice(2, 8);
await OtpModel.create({
  email,
  code: otp,
  expiresAt: new Date(Date.now() + 10 * 60 * 1000), // 10 mins
  isUsed: false
});

// Verify OTP
const record = await OtpModel.findOne({
  email,
  code,
  isUsed: false,
  expiresAt: { $gt: new Date() }
});

// Mark as used
await OtpModel.findByIdAndUpdate(otpId, { isUsed: true });
```

**Cleanup**: 
- Automatic with MongoDB TTL index (expiresAt)
- Or manual cleanup job every hour

### 8. Payment Collection

Records all payment transactions.

**Model**: `lib/db/models/Payment.ts`

```typescript
{
  _id: ObjectId,
  userId: ObjectId,                 // reference to User
  amount: Number,                   // in kobo (divide by 100 for naira)
  currency: String,                 // 'NGN', 'GHS', etc
  status: 'pending' | 'completed' | 'failed',
  paymentRef: String,               // Paystack reference
  planUpgrade: 'free' | 'pro',      // what plan purchased
  metadata: {
    email: String,
    description: String,
    orderId: String?
  },
  createdAt: Date,
  completedAt?: Date
}
```

**Indexes**:
- `userId` + `createdAt`
- `paymentRef` (unique)
- `status`

**Queries**:
```typescript
// Create payment record
await PaymentModel.create({
  userId,
  amount,
  currency: 'NGN',
  status: 'pending',
  paymentRef: generateRef(),
  planUpgrade: 'pro',
  metadata: {...}
});

// Update on webhook
await PaymentModel.findByIdAndUpdate(paymentId, {
  status: 'completed',
  completedAt: new Date()
});

// Get user's payment history
const payments = await PaymentModel
  .find({ userId })
  .sort({ createdAt: -1 })
  .limit(10);
```

## Migration & Backup

### MongoDB Atlas Features

- **Automated backups**: Daily snapshots
- **Point-in-time recovery**: Last 35 days
- **Multi-region replication**: Data redundancy
- **Atlas Data Lake**: Long-term archival

### Manual Backup

```bash
# Backup collection
mongodump --uri="mongodb+srv://user:pass@cluster.mongodb.net/dbname" \
  --collection=users \
  --out=./backups

# Restore collection
mongorestore --uri="mongodb+srv://user:pass@cluster.mongodb.net/dbname" \
  --collection=users ./backups/dbname/users.bson
```

## Monitoring & Maintenance

### Performance Metrics

Monitor in MongoDB Atlas:
1. **Ops/sec**: Queries per second
2. **Latency**: Query response time
3. **Connection count**: Active connections
4. **Replication lag**: Sync delay (if applicable)

### Index Optimization

Current indexes should be reviewed:
- Add indexes for frequently queried fields
- Remove unused indexes
- Monitor index performance

### Data Cleanup

Regular maintenance tasks:
```typescript
// Remove expired OTP codes
await OtpModel.deleteMany({
  expiresAt: { $lt: new Date() }
});

// Archive old outputs
await OutputModel.updateMany(
  { createdAt: { $lt: thirtyDaysAgo } },
  { archived: true }
);
```

## Best Practices

1. **Always await** database operations
2. **Use projections** to limit returned fields
3. **Implement pagination** for large datasets
4. **Index frequently queried fields**
5. **Cache query results** when appropriate
6. **Validate input data** before saving
7. **Use transactions** for multi-document operations
8. **Monitor slow queries** in MongoDB Atlas
